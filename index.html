Ralentissement du marché du numérique en France : Numeum appelle à la responsabilité collective

Ralentissement du marché du numérique en France

  • Croissance du marché : La croissance du marché numérique en France devrait ralentir à +1,8 % en 2025, contre +4,1 % fin 2024.
  • Secteurs touchés : Les ESN (Entreprises de Services du Numérique) et le conseil en technologies sont en repli (-2,1 % et -2,5 % respectivement), avec une baisse du taux d’occupation des équipes.
  • Résistance des éditeurs et plateformes cloud : Ces secteurs affichent une croissance de +8,2 %, portée par la migration vers le cloud (IaaS et PaaS) et des ajustements tarifaires.
  • IA générative : 48 % des entreprises travaillent sur des projets liés à l'IA générative, mais l'adoption est freinée par le manque de compétences (47 %) et la difficulté à identifier des cas d'usage à forte valeur ajoutée (47 %).
  • Emploi : Le secteur revient aux niveaux de 2022 avec environ 666 000 postes. Les perspectives de recrutement s’assombrissent, notamment pour les jeunes diplômés et alternants, avec 36 % des entreprises anticipant une baisse des offres.
  • Appel à la responsabilité collective : Numeum alerte sur le risque de décrochage de la France, qui ne pèse que 5,5 % du PIB dans le numérique, contre 10 % aux États-Unis. L'organisation appelle à maintenir les dispositifs fiscaux incitatifs à l’innovation et à investir massivement dans la transition numérique.
  • Transition numérique : Numeum plaide pour le développement de l’autonomie numérique européenne en matière de technologie et de compétences, soulignant l'urgence d'inverser la tendance pour éviter un décrochage sérieux de la France dans la compétition internationale.

"Comment j'ai vibe-codé une application de transport en deux heures"

Résumé de l'article :

L'article décrit l'expérience de développement d'un système de monitoring et d'alerting pour une ligne de transport parisienne en utilisant Cursor, un IDE intégrant l'IA. L'objectif était de créer une interface visuelle simple et un système d'alerting personnalisé pour la ligne P du Transilien.

Pour ce projet, l'auteur a choisi Cursor pour son intégration avec le modèle Claude 4 d'Anthropic, considéré comme le meilleur pour le développement logiciel. Avant de commencer avec Cursor, une première version de l'interface a été générée via Claude 4 Opus, en utilisant un prompt détaillé pour spécifier les fonctionnalités et le design souhaités.

L'interface a ensuite été améliorée avec Cursor en mode "Agent", permettant à l'IA d'agir de manière autonome pour modifier le code, ajouter des fichiers et exécuter des commandes bash. Pour la partie backend, l'auteur a utilisé l'API PRIM d'Ile-de-France Mobilités et a préféré désactiver la sélection automatique du modèle par Cursor pour utiliser des modèles plus efficaces ou moins chers, comme Claude 4 Sonnet Thinking pour les tâches complexes et Claude 4 Sonnet pour l'édition de code simple.

Le testing et le débogage ont été réalisés avec l'aide de l'agent de Cursor, qui a détecté et corrigé des problèmes dans la filtration des données de l'API. Le résultat final est une interface simple et fonctionnelle, affichant les prochains trains et alertant l'utilisateur en cas de suppressions ou de retards, via SMS et email.

L'article souligne les gains de temps significatifs offerts par le vibe-coding, réduisant un projet qui aurait pu prendre plusieurs jours à quelques heures. Cependant, il note aussi les limites de cette approche, comme la génération de code buggé ou halluciné, nécessitant une compréhension du code pour corrections. Malgré ces limites, les IDE intégrant l'IA comme Cursor augmentent la productivité et la créativité des développeurs.

Pourquoi les agents d'IA ont besoin d'une base de données opérationnelle

Résumé de l'article :

Les agents d'IA, assistants intelligents intégrés dans notre quotidien, doivent leur fluidité et leur réactivité à une base de données opérationnelle, infrastructure technique essentielle mais souvent invisible.

Ces agents, contrairement aux anciens chatbots rigides, apprennent et réagissent en temps réel, analysant des informations variées pour prendre des décisions autonomes. Ils s'appuient sur des modèles d'IA puissants comme GPT, nécessitant des flux de données continus et ultra-rapides.

Exemple concret : un agent d'IA dans une boutique en ligne utilise des données utilisateur, des catalogues produits, des stocks en temps réel, des avis clients et des documents non structurés pour offrir des recommandations personnalisées et précises.

Les bases analytiques classiques ne suffisent pas car elles sont conçues pour des analyses longues et non pour des échanges immédiats. Les agents d'IA nécessitent des réponses instantanées, comparables à une rame de métro qui doit constamment démarrer et s'arrêter.

Une base opérationnelle unifiée simplifie le processus, absorbant les pics d'activité, supportant divers formats de données et offrant un accès permanent aux informations cruciales. Cela améliore la performance, simplifie la maintenance et optimise l'expérience utilisateur.

Le choix d'une base opérationnelle est décisif pour l'avenir de l'IA, permettant des interactions perçues comme intelligentes, réactives et humaines. C'est une condition indispensable pour que l'IA tienne ses promesses d'une expérience numérique fluide et centrée sur l'humain.

Gemini CLI : Google dévoile un agent de code autonome entièrement gratuit

Gemini CLI : une alternative gratuite et puissante aux agents de code existants

Présentation et disponibilité Gemini CLI, présenté par Google le 25 juin, est un agent de code autonome intégré dans la console, disponible sous Windows, Linux et Mac. Il s'impose comme une alternative gratuite à Claude Code d'Anthropic et Codex CLI d'OpenAI.

Fonctionnalités principales - Édition et exécution de code : Clonage de dépôts, création de projets, déploiement d'applications, génération de code. - Gestion de versions : Création de branches Git, génération de pull requests, débogage automatique. - Analyse de code : Compréhension des dépendances, propositions d'améliorations contextualisées. - Intégrations multiples : Utilisable depuis le terminal, Slack, Microsoft Teams, et Gemini Code Assist dans Visual Studio Code. - Capacités avancées : Intégration native de Gemini 2.5 Pro avec une fenêtre de contexte de 1 million de tokens pour analyser de grandes bases de code.

Applications multimédias Grâce au MCP, Gemini CLI peut générer des contenus créatifs complexes, comme des vidéos, en utilisant les API natives de Google et les modèles de génération visuels (Veo, Imagen). Il peut être utilisé par des entrepreneurs, marketeurs, créateurs de contenu et professionnels pour produire des supports de communication, des présentations, des supports marketing et des contenus éditoriaux.

Politique de tarification - Gratuité : Contrairement à ses concurrents, Google propose Gemini CLI entièrement gratuit. - Limites d'utilisation : Jusqu'à 1 000 requêtes quotidiennes offertes (60 requêtes par heure). - Options avancées : Pour les entreprises, utilisation de clés API Vertex pour des configurations spécifiques.

Accessibilité et adoption Gemini CLI est accessible à tous dès le 25 juin, nécessitant uniquement un compte Google. Son adoption interne chez Google double chaque semaine parmi les ingénieurs.

Positionnement concurrentiel En offrant un outil gratuit, open source, multiplateforme et puissant, Google défie directement OpenAI et Anthropic, affirmant son ambition de devenir un acteur incontournable de l'IA pour les développeurs.

Avantages de l'usage du MCP avec un LLM en local - Économies de coûts : Évite la consommation excessive de tokens, coûteuse avec les modèles cloud. - Confidentialité : Données sensibles ne sont pas envoyées aux cloud providers, crucial pour les cas d'usage critiques. - Flexibilité : Peut fonctionner en environnement air-gap pour des exigences de sécurité maximales.

Installation et configuration - Prérequis : Python 3.13.3, Node.js. - Outils nécessaires : Tiny Agents d'Hugging Face (client MCP local) et Ollama (bibliothèque d'inférence locale). - Modèle recommandé : Qwen3 d'Alibaba, version 1.7B pour un PC classique sans GPU. - Configuration : - Création d'un dossier avec deux fichiers : agent.json (paramètres de l'agent) et PROMPT.md (instructions de base). - Exemple de configuration dans agent.json pour connecter l'agent à Ollama avec Qwen3:1.7b et ajouter un serveur MCP (Playwright).

Exécution et ajout de serveurs MCP additionnels - Commande d'exécution : tiny-agents run CheminVersVotreDossierAgent. - Exemple de prompt : Recherche de l'adresse LinkedIn d'une personne. - Ajout de serveurs MCP : Modification du fichier agent.json pour ajouter des serveurs MCP supplémentaires, comme l'accès aux fichiers locaux.

Le top 5 des meilleurs serveurs MCP du moment

Synthèse de l'article sur les serveurs MCP

Le Model Context Protocol (MCP) permet aux modèles d'IA de se connecter à diverses sources de données et outils de manière standardisée, fonctionnant sur une architecture client-serveur. Le client MCP (comme Claude Desktop ou Cursor) émet des requêtes, tandis que le serveur MCP, déployable en local ou dans le cloud, accède à des outils et sources de données externes.

Critères de sélection des serveurs MCP : - Intégration avec des outils de productivité (Google Workspace, Microsoft Office365). - Outils de développement logiciel (GitHub, GitLab). - Interaction et recherche web (Brave Search). - Popularité et sécurité (nombre d'étoiles sur GitHub, qualité du site).

Installation et configuration : - Les serveurs MCP peuvent être installés en local en suivant les instructions sur GitHub. - Configuration via un fichier JSON, comme mcp.json, pour spécifier les serveurs et leurs paramètres.

Serveurs MCP présentés : 1. Filesystem : Permet de lire, écrire et gérer des fichiers et répertoires. Installation simple via commande npx. 2. Playwright : Interagit avec les pages web, capture des écrans, génère du code de test. Installation facile via Cursor. 3. Browserbase : Extrait des données structurées du Web, navigue et identifie les formulaires. Nécessite une clé API et un Project_ID. 4. Stripe : Se connecte aux services de paiement de Stripe pour gérer clients, produits, factures, etc. Nécessite une clé secrète Stripe. 5. GitHub : Automatise des flux de travail et analyse des données de référentiels GitHub. Utilise un token d'accès personnel GitHub.

Chaque serveur MCP offre des fonctionnalités spécifiques et une facilité d'installation variable, répondant à des besoins précis en matière d'intégration et d'automatisation.
